# 无人机共享充电机巢管理系统开发文档

## 一、项目概述

### 1.1 项目背景

随着中国低空经济的快速发展，无人机应用场景日益丰富，但非共享机槽模式存在资源利用率低、无人机续航不足、运营成本高的问题，制约了低空经济规模化发展。本项目旨在搭建一套面向中国低空经济场景的无人机共享机槽管理系统，整合离散机槽资源为共享网络，通过智能调度破解续航瓶颈，助力低空经济发展。

### 1.2 项目目标

1. 基于高德 GIS 地图与北斗定位，实时呈现机槽位置、在线状态、占用情况及使用时长；

2. 支持三类无人机（固定路线、周期性、临时性）的差异化充电调度，预留深度图强化学习算法集成接口；

3. 实现多维度历史数据查询（机槽利用率、充电订单、调度记录）；

4. 构建稳定可靠的前后端架构，满足高并发场景下的调度需求；

5. 优先采用国内技术栈，保障系统安全性与合规性。

## 二、整体架构设计

### 2.1 架构分层

系统采用分层解耦的架构设计，分为感知层、接入层、平台层、应用层、数据层，适配物联网 + 共享经济的场景特性：

```mermaid
graph TD
    A[感知层] --> B[接入层]
    B --> C[平台层]
    C --> D[应用层]
    C --> E[数据层]
    F[运维监控层] --> B & C & D & E
    
    A[感知层]：无人机、充电机巢、北斗定位模块、传感器（电量/温度）
    B[接入层]：MQTT网关（EMQ X 国内版）、北斗定位接入服务
    C[平台层]：设备管理服务、充电调度服务、用户管理服务、算法适配服务
    D[应用层]：Web管理后台、移动端小程序、GIS监控页面
    E[数据层]：MySQL（业务数据）、Redis（缓存/实时状态）、MongoDB（日志/非结构化数据）
    F[运维监控层]：Prometheus+Grafana（国内镜像）、ELK（国内版）
```
### 2.2 技术栈选型（优先国内技术）

|层级|技术选型|选型理由|
|---|---|---|
|前端|Vue3 + Element Plus + 高德地图 JS API + 北斗定位 SDK|国内生态成熟，GIS 适配性强|
|后端|Spring Boot（国内版镜像） + EMQ X 国内版 + 北斗定位服务|稳定可靠，符合国内合规要求|
|存储|MySQL 8.0 + Redis 7.0 + MongoDB 7.0|适配结构化 / 非结构化数据存储|
|消息队列|RabbitMQ（国内镜像）|解耦设备消息与业务逻辑|
|监控|Prometheus（国内镜像） + Grafana|开源监控，适配国内部署|
## 三、前端开发方案

### 3.1 核心页面设计

#### 3.1.1 GIS 监控主页面

- **功能**：基于高德地图 JS API 展示机巢分布，北斗定位实时更新机巢位置；通过不同颜色标记机巢状态（在线空闲 / 在线占用 / 离线 / 故障），点击机巢弹窗展示详细信息（ID、充电功率、当前无人机信息、占用时长）。

- **响应式适配**：适配 PC 端（1920*1080）、平板端（768*1024）、移动端（375*667），移动端支持手势缩放地图。

- **交互组件**：

    - 地图工具栏：图层切换（卫星图 / 普通图）、搜索框（按机巢 ID / 位置搜索）、刷新按钮；

    - 状态筛选栏：按机巢状态、区域、充电功率筛选；

    - 实时数据面板：展示全网机巢在线率、平均利用率、当前充电订单数。

#### 3.1.2 订单管理页面

- **功能**：展示充电订单列表（订单 ID、无人机 ID、机巢 ID、充电时长、费用、状态），支持按时间、无人机类型、机巢 ID 筛选，支持订单详情查看与导出。

- **交互**：

    - 订单操作按钮：取消订单（未开始）、结束充电（充电中）、查看详情；

    - 批量操作：批量导出订单数据（Excel 格式）。

#### 3.1.3 无人机管理页面

- **功能**：展示无人机列表（无人机 ID、类型、所属企业、当前电量、飞行状态、绑定机巢），支持无人机信息编辑、绑定 / 解绑机巢。

- **差异化展示**：不同类型无人机用不同颜色标记（固定路线 - 蓝色、周期性 - 绿色、临时性 - 红色）。

#### 3.1.4 数据统计页面

- **功能**：用 ECharts 展示多维度统计数据：

    - 机巢利用率趋势图（日 / 周 / 月）；

    - 不同类型无人机充电订单占比饼图；

    - 全网无人机平均等待时间折线图；

    - 机巢故障统计柱状图。

### 3.2 前端技术实现要点

1. **高德地图集成**：

    - 引入高德地图 JS API（国内 CDN），配置开发者 Key；

    - 调用北斗定位服务获取机巢精准位置，同步至地图标记；

    - 实现机巢标记的实时更新（每 30 秒拉取一次机巢状态）。

2. **状态管理**：用 Pinia 管理全局状态（用户信息、地图状态、筛选条件），实现页面间状态共享。

3. **接口请求**：用 Axios 封装请求，统一处理请求拦截、响应拦截（token 验证、错误提示）。

4. **样式设计**：用 Tailwind CSS（国内镜像）实现响应式布局，结合 Element Plus 组件库快速搭建页面。

## 四、后端开发方案

### 4.1 服务架构

采用微服务架构（初期单体落地，后续拆分），核心服务包括：

1. **设备接入服务**：负责机巢、无人机的注册、认证、状态上报（基于 MQTT 协议），接收北斗定位数据，维护设备在线状态。

2. **充电调度服务**：处理充电请求，匹配空闲机巢，下发充电指令，对接算法适配服务获取调度方案。

3. **用户管理服务**：负责用户注册、登录、权限管理（角色：管理员、运维人员、企业用户），基于 JWT 实现认证。

4. **订单服务**：负责订单创建、状态更新、费用计算、订单查询。

5. **算法适配服务**：预留匹配算法集成接口，接收调度请求，调用算法服务返回匹配结果。

6. **数据统计服务**：负责统计数据的计算、存储、查询，支持报表生成。

### 4.2 核心功能实现

#### 4.2.1 设备接入与管理

- **设备注册**：机巢 / 无人机通过 MQTT 连接 EMQ X 网关，携带设备 ID、密钥进行认证，认证通过后注册至系统，分配唯一标识。

- **状态上报**：机巢每 30 秒上报一次状态（电量、温度、在线状态、当前无人机信息），无人机每 1 分钟上报一次电量、飞行状态、位置信息。

- **指令下发**：系统通过 MQTT 向机巢下发充电指令（启动 / 停止充电）、向无人机下发导航指令（前往指定机巢）。

#### 4.2.2 充电调度逻辑

1. **充电请求处理**：

    - 固定路线无人机：仅匹配航线两侧 500 米内的机巢，优先匹配能耗最低的机巢，提前 15 分钟预留机巢资源；

    - 周期性无人机：锁定预约时段的机巢资源，冲突时优先保障其预约权益；

    - 临时性无人机：紧急度≥4 级时优先抢占机巢资源，匹配充电功率≥1500W 的机槽。

2. **费用计算**：按充电时长 + 充电功率计算费用，支持企业套餐优惠（包月 / 包次）。

#### 4.2.3 北斗定位集成

调用北斗定位服务 API，获取机巢、无人机的精准位置，存储至数据库，同步至前端 GIS 地图。

### 4.3 后端技术实现要点

1. **MQTT 网关配置**：使用 EMQ X 国内版，配置设备认证规则，设置消息持久化，保障设备断连后消息不丢失。

2. **数据缓存**：用 Redis 存储机巢实时状态、无人机电量、订单缓存，提升查询效率。

3. **安全防护**：

    - 设备认证：采用设备密钥 + Token 双重认证，防止非法设备接入；

    - 接口鉴权：基于 JWT 实现接口权限控制，不同角色访问不同接口；

    - 数据加密：设备数据传输采用 MQTT over TLS，业务数据传输采用 HTTPS。

## 六、匹配算法接口预留

### 6.1 接口设计原则

1. **标准化**：采用 RESTful API 规范，接口参数、返回格式统一；

2. **低耦合**：算法服务与业务服务解耦，算法更新不影响核心业务；

3. **高可用**：设计兜底策略，算法服务异常时采用贪心算法兜底。

### 6.2 核心接口规范

#### 6.2.1 调度匹配接口

- **接口地址**：`POST /api/algorithm/schedule`

- **请求头**：

    - `Content-Type: application/json`

    - `Authorization: Bearer {token}`

- **请求参数**：

```json

{
  "drone_list": [
    {
      "drone_id": "DR001",
      "drone_type": 1, // 1-固定路线，2-周期性，3-临时性
      "current_battery": 30, // 当前电量（%）
      "location": "116.40717,39.90469", // 无人机当前位置（北斗定位，经纬度）
      "emergency_level": 1, // 紧急度：1-5，仅临时性无人机需传
      "route": "116.40717,39.90469;116.41717,39.91469" // 固定路线无人机的航线（经纬度列表）
    }
  ],
  "nest_list": [
    {
      "nest_id": "NT001",
      "location": "116.40717,39.90469", // 机巢位置（经纬度）
      "status": 1, // 1-空闲，2-占用
      "charge_power": 1500, // 充电功率（W）
      "available_time": "2026-02-25 12:00:00" // 机巢可用时间（周期性无人机预约）
    }
  ],
  "scene_params": {
    "weather": "normal", // 天气：normal/rain/wind
    "forbidden_area": ["116.40717,39.90469;116.41717,39.91469"] // 禁飞区（经纬度范围）
  }
}
```

- **返回参数**：

```json

{
  "code": 200,
  "msg": "匹配成功",
  "data": [
    {
      "drone_id": "DR001",
      "match_nest_id": "NT001",
      "match_reason": "距离最近，充电功率匹配",
      "estimated_charge_time": 60, // 预计充电时长（分钟）
      "estimated_arrival_time": 10 // 预计抵达机巢时间（分钟）
    }
  ],
  "backup_strategy": "greedy" // 兜底策略：greedy-贪心算法，km-匈牙利算法
}
```

#### 6.2.2 算法状态查询接口

- **接口地址**：`GET /api/algorithm/status`

- **请求头**：

    - `Authorization: Bearer {token}`

- **返回参数**：

```json

{
  "code": 200,
  "msg": "查询成功",
  "data": {
    "algorithm_name": "深度图强化学习算法",
    "status": "running", // running-运行中，error-异常，idle-空闲
    "version": "v1.0",
    "last_schedule_time": "2026-02-25 10:00:00",
    "schedule_count": 100 // 今日调度次数
  }
}
```

### 6.3 算法集成说明

1. 算法服务需部署为独立服务，通过 RESTful API 与业务系统交互；

2. 业务系统调用调度接口时，若算法服务超时（>100ms），自动触发兜底策略（贪心算法）；

3. 算法服务需返回标准化的匹配结果，包含匹配机巢 ID、预计时长等信息，便于业务系统处理。

## 七、部署与测试方案

### 7.1 部署方案

1. **本地开发环境**：

    - 前端：Vue3 项目，用 Vite 启动本地服务；

    - 后端：Spring Boot 项目，用 IDEA 启动；

    - 数据库：MySQL、Redis、MongoDB 用 Docker 容器启动（国内镜像）。

2. **测试环境**：

    - 用 Docker Compose 编排所有服务，部署至阿里云 ECS 服务器（国内节点）；

    - 配置 Nginx 反向代理，实现前后端分离部署。

3. **生产环境**：

    - 采用阿里云 Kubernetes（ACK）部署，实现服务自动扩缩容；

    - 配置 CDN 加速前端资源，提升访问速度；

    - 采用阿里云 RDS 托管 MySQL，保障数据安全性。

### 7.2 测试方案

#### 7.2.1 功能测试

1. **设备接入测试**：模拟机巢、无人机接入系统，验证状态上报、指令下发功能；

2. **调度测试**：模拟三类无人机的充电请求，验证调度逻辑、算法匹配结果；

3. **订单测试**：模拟充电订单的创建、支付、完成流程，验证费用计算、状态更新；

4. **GIS 测试**：验证高德地图、北斗定位的展示、更新功能。

#### 7.2.2 性能测试

1. **并发测试**：用 JMeter 模拟 1000 个无人机并发请求，验证系统吞吐量、响应时间；

2. **压力测试**：模拟 10000 个机巢、100000 个无人机的场景，验证系统稳定性；

3. **容错测试**：模拟机巢故障、网络断连，验证系统容错能力、自动重连功能。
