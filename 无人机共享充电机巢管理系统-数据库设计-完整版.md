### 1.2 核心实体关系说明

| 关系类型 | 主表 | 从表 | 关系描述 | 外键字段 |
|----------|------|------|----------|----------|
| 一对多 | enterprise | drone | 企业拥有多架无人机 | drone.enterprise_id |
| 一对多 | enterprise | charge_order | 企业产生多个订单 | charge_order.enterprise_id |
| 一对多 | nest_device | nest_slot | 机巢包含多个槽位 | nest_slot.nest_id |
| 一对多 | nest_device | charge_order | 机巢产生多个订单 | charge_order.nest_id |
| 一对多 | charge_order | order_fee_item | 订单包含多个费用项 | order_fee_item.order_id |
| 一对多 | charge_order | schedule_record | 订单关联调度记录 | schedule_record.execution_order_id |
| 多对多 | drone ↔ nest_device | charge_order | 无人机与机巢通过订单关联 | 关联表：charge_order |

## 二、核心数据表结构设计

### 2.1 企业信息表 (enterprise)

存储入驻系统的企业用户基本信息。

| 字段名 | 数据类型 | 是否为空 | 默认值 | 说明 |
|--------|----------|----------|--------|------|
| id | bigint unsigned | 否 | AUTO_INCREMENT | 主键ID |
| enterprise_code | varchar(32) | 否 | - | 企业编码，唯一 |
| enterprise_name | varchar(128) | 否 | - | 企业名称 |
| enterprise_type | tinyint | 否 | 1 | 企业类型：1物流企业/2巡检企业/3测绘企业/4其他 |
| contact_name | varchar(64) | 是 | NULL | 联系人姓名 |
| contact_phone | varchar(20) | 是 | NULL | 联系人电话 |
| contact_email | varchar(128) | 是 | NULL | 联系人邮箱 |
| credit_code | varchar(32) | 是 | NULL | 统一社会信用代码 |
| business_license_url | varchar(256) | 是 | NULL | 营业执照图片URL |
| province_code | varchar(12) | 是 | NULL | 省份编码 |
| city_code | varchar(12) | 是 | NULL | 城市编码 |
| district_code | varchar(12) | 是 | NULL | 区县编码 |
| address | varchar(256) | 是 | NULL | 详细地址 |
| status | tinyint | 否 | 0 | 状态：0待审核/1正常/2禁用/3注销 |
| account_balance | decimal(12,2) | 否 | 0.00 | 账户余额 |
| credit_score | int | 否 | 100 | 信用评分(0-100) |
| created_at | datetime | 否 | CURRENT_TIMESTAMP | 创建时间 |
| updated_at | datetime | 否 | CURRENT_TIMESTAMP | 更新时间 |
| created_by | bigint unsigned | 是 | NULL | 创建人ID |
| updated_by | bigint unsigned | 是 | NULL | 更新人ID |
| remark | varchar(512) | 是 | NULL | 备注 |
| deleted | tinyint | 否 | 0 | 删除标记：0未删除/1已删除 |

**索引设计：**
```sql

UNIQUE KEY `uk_enterprise_code` (`enterprise_code`),
INDEX `idx_status` (`status`),
INDEX `idx_enterprise_type` (`enterprise_type`),
INDEX `idx_created_at` (`created_at`),
INDEX `idx_deleted` (`deleted`)
```
